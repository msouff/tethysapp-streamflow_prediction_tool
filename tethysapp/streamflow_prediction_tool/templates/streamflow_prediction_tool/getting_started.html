¡Bienvenido!

¿Es esta su primera vez utilizando la Herramienta de Predicción de Caudales o simplemente necesita refrescar su memoria? De cualquier manera, esta página contiene la información que necesita para comenzar a usar esta herramienta.

Guía

**Los pasos deben ser completados en el mismo orden dado para que la herramienta funcione correctamente**

Para facilitar la aplicación de los pasos, sugerimos que abra esta herramienta en otra ventana, usando una ventana para aplicar los pasos y la otra para leer las instrucciones.

  1. Ajutes Principales

    2. Antes que nada, debe asegurarse que los parametros en la página de ajustes principales de la herramienta son los correctos. Vaya a http://localhost:8000/admin/tethys_apps/tethysapp y seleccione 'Herramienta de Predicción de Caudales'

    • App ID

      • Este es el ID asignado a su herramienda. Es usado para ciertas tareas y se asegura que los datos producidos son únicos.

    • Server Folders

      • Estos son los directorios locales donde se guardarán los pronósticos.

      NOTA: Este servidor local es el servido donde está instalada la herramienta. Por ejemplo, For example, una instancia de esta herramienta existe en tethys.byu.edu y esta alojada en servidores de Brigham Young University. Si instaló Tethys en su propia computadora, entoces su computadora sería su servidor local. Cualquiera que sea su servidor, si usted no tiene acceso a él NO CAMBIE ESTOS DIRECTORIOS.

      Hay dos campos correspondientes a diferentes directorios en su servidor local: uno para los pronósticos de caudales y otro para los pronósticos historicos simulados. Cada directorio debe tener permisos de lectura y escritura. Para cambiar los permisos, vaya a el directorio y haga click derecho en él y seleccione "Obtener Información" (Usuarios de Mac) o "Propiedades" (Usuarios de Linux o Windows). Seleccione el tab con el encabezado "permisos" y cambie los permisos a permisos de lectura y escritura. Si está utilizando la terminal, simplemente escriba (asumiendo que el usuario del servidor es apache): #chown -r apache:apache [directorio]; chmod -R u+rx [directorio].

  2. Agregar un Servidor Geoespacial

    3. Una vez los ajustes principales han sido establesidos, es hora de agregar un servidor geoespacial. En la barra de navegación a la izquierda bajo el encabezado Agregar haga click en la opción de Agregar un Servidor Geoespacial

    Verá la siguiente pantalla:



    En esta página usted agregará un servidor geoespacial (Geoserver) con el que estará trabahando. Puede agregar varios servidores. Estos pueden ser administrados luego (editados/borrados) en la option de Administración de Servidor Geoespacial en la barra de navegación a la izquierda bajo el encabezado de Administración. Por cada servidor geoespacial que agregue, todos los campos mostrados deben ser completados. Una descripción de cada campo es dada aquí:

    • Nombre de Servidor Geoespacial

      • Como puede agregar más de un servidor, un nombre único debe ser dado para ayudar a reconocer cada servidor. Este nombre puede ser cualquier nombre que desee.

    • Url del Servidor

      • Este es el URL específico de su servidor geoespacial (Geoserver). NOTA: Este URL debe terminar en "/geoserver".

    • Nombre de Usuario

      • Este es el usuario asociado con su servidor geoespacial (Geoserver).

    • Contraseña

      • Esta es la contraseña asociada con su servidor geoespacial (geoserver).

  3. Agregar una Base de Datos

    4. En la barra de navegación a la izquierda bajo el encabezado de Agregar haga click en la opción de Agregar Base de Datos.

    Verá la siguiente pantalla:



    En esta página agregará la base de datos con la que trabajará. Puede agregar más de una base de datos. Estas pueden ser administradas luego (editadas/borradas) en la opción de Administración de Base de Datos localizada en la barra de navegación a la izquierda bajo el encabezado de Administración. Por cada base de datos que agregue, todos los campus mostrados deben ser completados. Una descripción de cada campo es dada aquí:

    • Nombre de la Base de Datos

      • Como puede agregar más de una base de datos, un nombre único debe ser dado para ayudar a reconocer cada base de datos. Este nombre puede ser cualquier nombre que desee.

    • Tipo de Base de Datos

      • Este es el tipo de base de datos. Hay dos opciones: CKAN o HydroShare. CKAN es la única base de datos external funcional en este momento. También puede utilizar su servidor local como base de datos. Si no conoce CKAN, puede aprender más de esta aquí.

    • API Endpoint de la Base de Datos

      • Este es el URL de su base de datos. NOTA: Este URL debe terminar en "/api/3/action".

    • API Key de la Base de Datos

      • Este es la llave asociada con su API endpoint.

  4. Agregar una Cuenca

    5. Ahora llegamos a la parte esencial de esta herramienta: agregar cuencas para que puedan ser visualizadas. en la barra de navegación a la izquierda bajo el encabezado Agregar haga click en la opción de Agregar una Cuenca.

    Verá la siguiente pantalla:



    En esta página agregará las cuencas con las que trabajará. Puede agregar más de una cuenca. Estas pueden ser administradas luego (editadas/borradas) en la opción de Administración de Cuencas localizada en la barra de navegación a la izquierda bajo el encabezado de Administración. El formulario para agregar cuencas es dinámico y el número de campos requeridos dependerá de su selección en campos previos. Una descripción de cada campo es dada aquí:

    • Nombre del Area de Interés

      • Como puede agregar más de un área de interés, un nombre único debe ser dado para ayudar a reconocer cada área de interés. Este nombre puede ser cualquier nombre que desee.

    • Nombre de la Cuenca

      • Como puede agregar más de una cuenca, un nombre único debe ser dado para ayudar a reconocer cada cuenca. Este nombre puede ser cualquier nombre que desee.

    • Seleccione una base de datos

      • La opción por default es su servidor local. Esta opción no está asociada a ningún otro campo. Las demmás opciones que aparezcan en este campo corresponden a la o las bases de datos agregadas previamentes.

      • Local Server (local)
        • Nombre del Area de Interes en la Base de Datos

          • Este es el nombre del área de interés en la base de datos y por ende puede direfir del nombre dado en la opción de "Nombre del Aréa de Interés.

        • Nombre de la Cuenca en la Base de Datos

          • Este es el nombre de la cuenca en la base de datos y por ende puede direfir del nombre dado en la opción de "Nombre de la Cuenca.

      • Otra base de datos que no es la default

        • Si esta opción es seleccionada, los siguientes campos aparecerán dinámicamente:

        • GloFAS-RAPID Archivos de Entrada (Opcional)

          • Puede subir archivos de entrada para su cuenca por esta vía.

    • Seleccione un Servidor Geoespacial

      • Hay does opciones asociadas con este campo.

      • Subir Shapefile?
        • Si

          • Puede subir un shapefile al servidor geoespacial seleccionado.

          • Shapefile con Red Fluvial

            • Esta es la red fluvial y es un campo REQUERIDO.

          • Shapefile con Cuencas (Opcional)
          • Shapefile con estaciones de observación (Opcional)
          • Shapefile con estaciones AHPS (Opcional)
        • No

          • Esta opción es para capas que ya han sido agregadas anteriormente al servidor geoespacial seleccionada. El nombre de la capa debe sere especificado como se encuentra en el servidor geoespacial (Geoserver).

          • Shapefile con Red Fluvial

            • Esta es la red fluvial y es un campo REQUERIDO.

          • Shapefile con Cuencas (Opcional)
          • Shapefile con estaciones de observación (Opcional)
          • Geoserver Mapa con Inundaciones Historicas (Opcional)

            • Nota: Esta no es una capa. Es un grupo de capas.

          • Shapefile con estaciones AHPS (Opcional)
